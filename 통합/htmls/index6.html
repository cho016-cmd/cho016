실시간 버스 도착 알림 시스템 개발

연구자: 김강현

1. 연구 주제

공공데이터 API를 활용한 실시간 버스 도착 알림 시스템 개발

이 대시보드는 '실시간 버스 도착 알림 시스템' 개발 연구 보고서를 대화형으로 탐색할 수 있도록 재구성한 것입니다. 상단의 탭을 클릭하여 연구의 동기, 활용된 기술, 개발 과정 및 결론을 확인해 보세요.

2. 연구 개요

본 연구가 왜 필요한지, 무엇을 목표로 하는지, 그리고 어떤 핵심 질문에서 시작되었는지에 대한 개요입니다.

가. 연구 동기 및 필요성

매일 버스를 이용하지만, 버스 도착 시간은 예상과 다른 경우가 많습니다. 스마트폰 앱으로 확인 가능하지만, 계속 화면을 확인해야 하는 불편함이 있습니다. 이러한 불편함을 해결하고자, 버스가 곧 도착할 때 '능동적으로' 알림을 주는 시스템을 개발하여 시간 낭비를 줄이고 대중교통 이용 경험을 개선하고자 합니다.

나. 연구 목적

공공데이터 API를 통해 수집한 실시간 버스 위치 정보를 활용하여, 사용자가 지정한 버스가 정류장에 도착하기 전에 미리 알려주는 웹 기반의 실시간 알림 시스템을 개발하는 것을 최종 목적으로 합니다.

다. 연구 문제

  • 공공데이터 API를 통해 어떻게 실시간 버스 도착 정보를 안정적으로 수신할 수 있는가?
  • 사용자에게 가장 효과적인 버스 도착 알림 방식(시각적, 청각적)은 무엇이며, 이를 웹 환경에서 어떻게 구현할 것인가?
  • 사용자 편의성을 고려한 최적의 알림 시점(예: 도착 3분 전)을 결정하고 시스템에 적용하는 방안은 무엇인가?

3. 활용 기술

이 연구를 수행하기 위해 활용된 핵심 기술입니다. 아래 버튼을 클릭하여 각 기술의 역할을 확인해 보세요.

공공데이터 API (REST API)

공공데이터포털에서 제공하는 '버스도착정보조회 서비스' API를 활용합니다. 정해진 형식으로 요청을 보내면 실시간 버스 운행 정보를 응답으로 받아오는 기술을 이해하고 활용합니다.

Python (핵심 로직)

API를 호출하고, 응답으로 받은 복잡한 XML 형식의 데이터를 파싱(parsing)하여 '남은 시간', '차량 번호' 등 필요한 정보만 추출하는 핵심 데이터 처리 로직을 Python으로 구현했습니다.

Streamlit (시각화)

Python 코드만으로 인터랙티브 웹사이트를 빠르고 쉽게 구축할 수 있는 Streamlit 라이브러리를 사용했습니다. 이를 통해 API로 받은 데이터를 사용자 친화적인 UI로 시각화하는 웹사이트를 구현했습니다.

EXE 패키징 (알림 기능)

핵심 알림 기능(5분 전 알림)을 구현한 Python 스크립트를 사용자가 별도 설치 없이 쉽게 실행할 수 있도록 .exe 실행 파일로 패키징했습니다. 이를 위해 PyInstaller와 같은 도구를 활용했습니다.

주요 검색 키워드

#JavaScript_Fetch_API #Web_Notifications_API #Service_Worker_Push #실시간_데이터_처리

4. 개발 과정 및 결과

연구를 수행한 절차 및 구체적인 최종 결과물입니다. 하위 탭을 선택하여 세부 내용을 확인하세요.

가. 최종 결과물

본 연구를 통해 두 가지 주요 결과물을 개발했습니다. 하나는 핵심 알림 기능에 중점을 둔 Python 프로그램이며, 다른 하나는 데이터 시각화에 중점을 둔 Streamlit 웹사이트입니다.

1. Python 알림 프로그램 (.exe)

사용자가 터미널에서 원하는 정류장과 버스를 입력하면, 실시간으로 남은 시간을 갱신하여 보여줍니다. 버스가 5분 이내로 도착하면 **알림음을 발생시켜** 사용자에게 즉각적으로 알려주는 핵심 기능을 수행합니다.

2. Streamlit 시각화 사이트

알림 기능 대신, API로 받은 버스 도착 정보를 사용자가 한눈에 보기 쉽게 **시각화하는 대시보드**입니다. Python 코드만으로 완성도 높은 인터랙티브 웹 UI를 구축하여 데이터를 효과적으로 전달합니다.

나. 개발 절차

1단계
요구사항 분석 및 설계

핵심 기능을 정의하고 Python 로직 및 Streamlit UI를 설계했습니다.

2단계
개발 환경 구축

Python 환경을 설정하고 공공데이터 API 활용을 신청했습니다.

3단계
핵심 기능 개발

API 호출, 데이터 파싱, 알림 발생 로직 및 Streamlit 웹 UI를 구현했습니다.

4단계
테스트 및 개선

.exe 파일과 Streamlit 사이트를 테스트하고 버그를 수정하여 완성했습니다.

다. 실제 주요 활동 일정

원본의 계획을 실제 수행한 내역 기반으로 간트 차트를 시각화했습니다. 막대에 마우스를 올리면 세부 기간을 볼 수 있습니다.

5. 결론 및 고찰

본 연구를 통해 개발된 실시간 버스 도착 알림 시스템은 사용자가 더 이상 스마트폰 화면을 계속 주시할 필요 없이, 다른 활동을 하다가 버스 도착 시간에 맞춰 준비할 수 있도록 도와줍니다. 🚌

이를 통해 바쁜 학생과 직장인들의 시간 낭비를 줄여주고, 버스를 놓칠지 모른다는 불안감을 해소하여 대중교통 이용 경험의 질을 크게 향상시킬 수 있습니다.

"궁극적으로는 이러한 편리한 시스템이 대중교통 이용을 활성화하는 데 긍정적인 영향을 미쳐, 교통 혼잡 완화 및 환경 보호와 같은 사회적 가치 창출에도 기여할 수 있을 것으로 기대됩니다. ✨"

가. 고찰 (한계점 및 향후 계획)

프로젝트를 완성하며 몇 가지 한계점과 향후 개선 방향을 도출할 수 있었습니다.

  • 기능 분리의 한계: 현재 핵심 알림 기능은 .exe 프로그램에, 시각화 기능은 Streamlit 웹사이트에 분리되어 있어 사용자가 두 가지를 동시에 경험하기 어렵습니다.
  • 플랫폼 의존성: .exe 파일은 Windows 환경에서만 작동하며, 알림음을 사용한 방식은 웹 표준 알림에 비해 범용성이 떨어집니다.

향후 계획으로는 이 두 가지 기능을 하나의 완성도 높은 웹 애플리케이션으로 통합하는 것입니다.

Streamlit 또는 다른 웹 프레임워크(React, Vue 등)를 기반으로, 백그라운드에서도 작동하는 '웹 푸시 알림' 또는 '서비스 워커' 기술을 도입하여, 별도 설치 없이 누구나 웹에서 실시간 알림을 받을 수 있도록 고도화할 것입니다.
